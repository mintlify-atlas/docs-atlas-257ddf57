---
title: "Transformers API"
description: "HuggingFace-compatible AutoModel classes with built-in IPEX-LLM quantization and optimization."
---

The `ipex_llm.transformers` module provides drop-in replacements for HuggingFace Transformers `AutoModel` classes. You load a model exactly as you would with `transformers`, but pass additional parameters to enable low-bit quantization and Intel hardware optimizations.

<Note>
All AutoModel variants listed below share the same extended `from_pretrained` signature. The examples on this page use `AutoModelForCausalLM`, but the API is identical for `AutoModel`, `AutoModelForSpeechSeq2Seq`, `AutoModelForSeq2SeqLM`, `AutoModelForSequenceClassification`, `AutoModelForMaskedLM`, `AutoModelForQuestionAnswering`, `AutoModelForNextSentencePrediction`, `AutoModelForMultipleChoice`, and `AutoModelForTokenClassification`.
</Note>

## Supported AutoModel classes

| Class | Typical use case |
|---|---|
| `AutoModelForCausalLM` | Text generation, chat |
| `AutoModel` | General-purpose, custom heads |
| `AutoModelForSpeechSeq2Seq` | Speech-to-text (Whisper) |
| `AutoModelForSeq2SeqLM` | Seq2seq (Flan-T5, etc.) |
| `AutoModelForSequenceClassification` | Text classification |
| `AutoModelForMaskedLM` | Masked language modeling |
| `AutoModelForQuestionAnswering` | Extractive QA |
| `AutoModelForNextSentencePrediction` | NSP tasks |
| `AutoModelForMultipleChoice` | Multiple-choice tasks |
| `AutoModelForTokenClassification` | NER, tagging |

## `AutoModelForCausalLM`

```python
from ipex_llm.transformers import AutoModelForCausalLM
```

### `from_pretrained`

```python
AutoModelForCausalLM.from_pretrained(*args, **kwargs)
```

Load a model from a local directory or the HuggingFace Hub. Accepts all standard HuggingFace `from_pretrained` arguments plus the IPEX-LLM extensions below.

#### Parameters

<ParamField path="pretrained_model_name_or_path" type="string" required>
  Path to a local model directory or a HuggingFace model ID (e.g. `"meta-llama/Llama-2-7b-chat-hf"`).
</ParamField>

<ParamField path="load_in_4bit" type="boolean" default="False">
  When `True`, loads linear layer weights in symmetric INT4 for regular FP16/BF16/FP32 models, or in asymmetric INT4 for GPTQ models. Mutually exclusive with `load_in_low_bit`.
</ParamField>

<ParamField path="load_in_low_bit" type="string">
  Quantization format for linear weights. Accepted values:

  `sym_int4`, `asym_int4`, `sym_int5`, `asym_int5`, `sym_int8`, `nf3`, `nf4`, `fp4`, `fp8`, `fp8_e4m3`, `fp8_e5m2`, `fp6`, `fp6_k`, `fp16`, `bf16`, `gguf_iq2_xxs`, `gguf_iq2_xs`, `gguf_iq1_s`, `gguf_q4k_m`, `gguf_q4k_s`.

  See [Quantization Formats](/api/quantization-formats) for guidance on choosing a format.
</ParamField>

<ParamField path="optimize_model" type="boolean" default="True">
  Whether to apply additional kernel-level optimizations to the loaded model (attention, MLP fusion, etc.). Disable if you need to inspect or modify the model graph before optimizing.
</ParamField>

<ParamField path="modules_to_not_convert" type="string[]" default="None">
  List of module names (matched against `nn.Module` names) that should be skipped during quantization. Useful for keeping sensitive layers (e.g. `lm_head`) in full precision.
</ParamField>

<ParamField path="cpu_embedding" type="boolean" default="False">
  When `True`, the `Embedding` layer is kept on CPU even when the rest of the model runs on GPU. Required on some GPU configurations where embedding lookup on device causes errors.
</ParamField>

<ParamField path="speculative" type="boolean" default="False">
  Enable speculative decoding support. Requires a compatible draft model to be configured separately.
</ParamField>

<ParamField path="imatrix" type="string" default="None">
  Path to an importance matrix file (produced by `llama.cpp`). Used to improve quantization quality for GGUF-based quantization methods such as `gguf_iq2_xxs`.
</ParamField>

<ParamField path="model_hub" type="string" default="huggingface">
  Model hub to download from. Options: `"huggingface"` or `"modelscope"`.
</ParamField>

<ParamField path="embedding_qtype" type="string" default="None">
  Quantization type for `nn.Embedding` layers. Accepted values: `"q2_k"`, `"q4_k"`. When `None`, embeddings are not quantized.
</ParamField>

<ParamField path="mixed_precision" type="boolean" default="False">
  When `True` and `load_in_low_bit` is `"sym_int4"` or `"asym_int4"`, the `lm_head` layer uses `sym_int8` instead of INT4 to improve output quality.
</ParamField>

<ParamField path="pipeline_parallel_stages" type="integer" default="1">
  Number of GPUs to use for pipeline parallelism. Set to a value greater than `1` to split the model across multiple Intel GPUs.
</ParamField>

<ParamField path="torch_dtype" type="string | torch.dtype">
  Base dtype for non-quantized parameters. Commonly `"auto"`, `torch.float16`, or `torch.bfloat16`.
</ParamField>

#### Returns

<ResponseField name="model" type="PreTrainedModel">
  An optimized model instance with the same interface as a standard HuggingFace model. The returned model also exposes a `save_low_bit(save_dir)` method for serialization.
</ResponseField>

### `from_gguf`

```python
AutoModelForCausalLM.from_gguf(
    fpath,
    optimize_model=True,
    cpu_embedding=False,
    low_bit="sym_int4"
)
```

Load a GGUF model file and convert it to an IPEX-LLM model along with a HuggingFace tokenizer.

<ParamField path="fpath" type="string" required>
  Absolute or relative path to the `.gguf` model file.
</ParamField>

<ParamField path="optimize_model" type="boolean" default="True">
  Whether to apply kernel-level optimizations after conversion.
</ParamField>

<ParamField path="cpu_embedding" type="boolean" default="False">
  Keep the embedding layer on CPU. Set to `True` when running on GPU.
</ParamField>

<ParamField path="low_bit" type="string" default="sym_int4">
  Target quantization format for the converted model.
</ParamField>

#### Returns

<ResponseField name="(model, tokenizer)" type="tuple">
  A tuple of `(ipex_llm model, HuggingFace tokenizer)`.
</ResponseField>

---

### `load_low_bit`

```python
AutoModelForCausalLM.load_low_bit(
    pretrained_model_name_or_path,
    *model_args,
    **kwargs
)
```

Load a previously saved low-bit model checkpoint.

<ParamField path="pretrained_model_name_or_path" type="string" required>
  Path to the directory containing the saved low-bit checkpoint (written by `model.save_low_bit()`).
</ParamField>

<ParamField path="optimize_model" type="boolean" default="True">
  Whether to apply kernel-level optimizations after loading.
</ParamField>

#### Returns

<ResponseField name="model" type="PreTrainedModel">
  The loaded and optimized model instance.
</ResponseField>

## Usage examples

<Tabs>
  <Tab title="INT4 (GPU)">
    ```python
    from ipex_llm.transformers import AutoModelForCausalLM
    from transformers import AutoTokenizer

    model_path = "meta-llama/Llama-2-7b-chat-hf"

    model = AutoModelForCausalLM.from_pretrained(
        model_path,
        load_in_4bit=True,
        cpu_embedding=True,   # recommended for GPU
        optimize_model=True,
    )
    model = model.half().to("xpu")

    tokenizer = AutoTokenizer.from_pretrained(model_path)

    inputs = tokenizer("Once upon a time", return_tensors="pt").to("xpu")
    output = model.generate(**inputs, max_new_tokens=100)
    print(tokenizer.decode(output[0], skip_special_tokens=True))
    ```
  </Tab>
  <Tab title="NF4 (CPU)">

    ```python
        model_path,
        load_in_low_bit="nf4",
        optimize_model=True,
    )

    tokenizer = AutoTokenizer.from_pretrained(model_path)

    inputs = tokenizer("Explain quantum entanglement:", return_tensors="pt")
    output = model.generate(**inputs, max_new_tokens=200)
    print(tokenizer.decode(output[0], skip_special_tokens=True))
    ```
  </Tab>
  <Tab title="BF16 (GPU)">
    ```python
    from ipex_llm.transformers import AutoModelForCausalLM
    from transformers import AutoTokenizer
    import torch

    model_path = "Qwen/Qwen2-7B-Instruct"

    model = AutoModelForCausalLM.from_pretrained(
        model_path,
        load_in_low_bit="bf16",
        optimize_model=True,
        cpu_embedding=True,

    ```
  </Tab>
  <Tab title="Save & reload">
    ```python
    from ipex_llm.transformers import AutoModelForCausalLM
    from transformers import AutoTokenizer

    # Load and quantize
    model = AutoModelForCausalLM.from_pretrained(
        "meta-llama/Llama-2-7b-chat-hf",
        load_in_low_bit="sym_int4",
    )
    model.save_low_bit("./llama2-7b-int4")

    # Later: reload without re-quantizing
    model = AutoModelForCausalLM.load_low_bit("./llama2-7b-int4")
    ```

## Using HuggingFace tokenizers

IPEX-LLM models are fully compatible with standard HuggingFace tokenizers. Always load the tokenizer from the original model path — no special tokenizer class is required.

```python
from transformers import AutoTokenizer

# Standard HuggingFace tokenizer — no changes needed
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-chat-hf")

inputs = tokenizer("Hello, world!", return_tensors="pt")
outputs = model.generate(**inputs, max_new_tokens=50)
response = tokenizer.decode(outputs[0], skip_special_tokens=True)
```

<Tip>
For models that require `trust_remote_code=True` (e.g. ChatGLM, Qwen), pass the same flag to both the model and the tokenizer loader.
</Tip>

## Low-bit format reference

| Value | Bits | Description |
|---|---|---|
| `sym_int4` | 4 | Symmetric INT4 (default for `load_in_4bit=True`) |
| `asym_int4` | 4 | Asymmetric INT4 |
| `sym_int5` | 5 | Symmetric INT5 |
| `asym_int5` | 5 | Asymmetric INT5 |
| `sym_int8` | 8 | Symmetric INT8 |
| `nf4` | 4 | 4-bit NormalFloat (QLoRA-style) |
| `fp4` | 4 | 4-bit floating point |
| `fp8` | 8 | 8-bit floating point |
| `fp8_e4m3` | 8 | FP8 E4M3 variant |
| `fp8_e5m2` | 8 | FP8 E5M2 variant |
| `fp6` | 6 | 6-bit floating point |
| `fp16` | 16 | Half precision |
| `bf16` | 16 | Brain float 16 |

See [Quantization Formats](/api/quantization-formats) for detailed tradeoffs and memory estimates.
