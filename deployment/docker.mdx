---

IPEX-LLM provides Docker images that bundle all required dependencies — GPU drivers, oneAPI runtimes, and the IPEX-LLM library — so you can run LLM inference and serving on Intel GPUs without manual environment setup.

## Available images

| Image | Use case |
|---|---|
| `intelanalytics/ipex-llm-serving-xpu:latest` | GPU inference & serving (PyTorch, vLLM, FastChat) |
| `intelanalytics/ipex-llm-inference-cpp-xpu:latest` | llama.cpp, Ollama, and Open WebUI on GPU |
| `intelanalytics/ipex-llm-serving-cpu:latest` | CPU inference & serving via vLLM or FastChat |
| `intelanalytics/ipex-llm-finetune-qlora-xpu:latest` | GPU fine-tuning with QLoRA/LoRA |

<Note>
The Windows + WSL + Docker solution only supports Arc series dGPUs. For Windows users with an MTL iGPU, install directly via pip in a Miniforge prompt instead.
</Note>

## Python GPU inference

### Prerequisites

Install Docker by following the [official Docker guide](https://docs.docker.com/get-started/).

For Windows, you additionally need WSL2 enabled and Docker Desktop configured with WSL2 integration:

1. Install [Docker Desktop for Windows](https://docs.docker.com/desktop/install/windows-install/) and restart.
2. Install [WSL2](https://docs.microsoft.com/en-us/windows/wsl/install).
3. In Docker Desktop, go to **Settings → Resources → WSL integration**, enable Ubuntu, and click **Apply & restart**.

### Pull the image

```bash
docker pull intelanalytics/ipex-llm-serving-xpu:latest
```

### Start the container

<Tabs>
  <Tab title="Linux">
    ```bash
    export DOCKER_IMAGE=intelanalytics/ipex-llm-serving-xpu:latest
    export CONTAINER_NAME=my_container
    export MODEL_PATH=/llm/models  # change to your local model directory

    docker run -itd \
      --net=host \
      --device=/dev/dri \
      --memory="32G" \
      --name=$CONTAINER_NAME \
      --shm-size="16g" \
      -v $MODEL_PATH:/llm/models \
      --entrypoint /bin/bash \
      $DOCKER_IMAGE
    ```
  </Tab>
  <Tab title="Windows WSL">
    ```bash
    export DOCKER_IMAGE=intelanalytics/ipex-llm-serving-xpu:latest
    export CONTAINER_NAME=my_container
    export MODEL_PATH=/llm/models  # change to your local model directory

    sudo docker run -itd \
      --net=host \
      --privileged \
      --device /dev/dri \
      --memory="32G" \
      --name=$CONTAINER_NAME \
      --shm-size="16g" \
      -v $MODEL_PATH:/llm/llm-models \
      -v /usr/lib/wsl:/usr/lib/wsl \
      --entrypoint /bin/bash \
      $DOCKER_IMAGE
    ```
  </Tab>
</Tabs>

Enter the running container:

```bash
docker exec -it $CONTAINER_NAME bash
```

### Verify GPU access

Run `sycl-ls` inside the container to confirm the GPU is mapped correctly:

```bash
root@arda-arc12:/# sycl-ls
[opencl:acc:0] Intel(R) FPGA Emulation Platform for OpenCL(TM), Intel(R) FPGA Emulation Device ...
[opencl:cpu:1] Intel(R) OpenCL, 13th Gen Intel(R) Core(TM) i9-13900K ...
[opencl:gpu:2] Intel(R) OpenCL Graphics, Intel(R) Arc(TM) A770 Graphics ...
[ext_oneapi_level_zero:gpu:0] Intel(R) Level-Zero, Intel(R) Arc(TM) A770 Graphics ...
```

<Tip>
Run the environment check script to verify your IPEX-LLM installation and GPU runtime:

```bash
cd /ipex-llm/python/llm/scripts
bash env-check.sh
```
</Tip>

### Recommended environment variables

Set these before running inference for best performance:

```bash
# Disable XETLA — only Intel Data Center GPU Max Series supports it.
# Recommended for Arc A-Series and Flex Series.
export USE_XETLA=OFF

# Enable immediate command lists for Level Zero. Improves performance
# on Arc A-Series and Data Center GPU Max Series (kernel-dependent).
export SYCL_PI_LEVEL_ZERO_USE_IMMEDIATE_COMMANDLISTS=1

# Enable persistent compiled code cache (already set by default in Docker images).
export SYCL_CACHE_PERSISTENT=1

# Fuse SDP ops to reduce memory accesses.
# Recommended for Intel Data Center GPU Max Series.
export ENABLE_SDP_FUSION=1
```

### Run inference

Start an interactive chat session with a mounted model:

```bash
cd /llm
python chat.py --model-path /llm/models/Llama-2-7b-chat-hf
```

Run a generation example directly:

```bash
cd /examples/llama2
python ./generate.py \
  --repo-id-or-model-path /llm/models/Llama-2-7b-chat-hf \
  --prompt "What is AI?" \
  --n-predict 128
```

## llama.cpp and Ollama

The `ipex-llm-inference-cpp-xpu` image provides llama.cpp and Ollama with Intel GPU support, plus an optional Open WebUI frontend.

### Pull the image

```bash
docker pull intelanalytics/ipex-llm-inference-cpp-xpu:latest
```

### Start the container

<Tabs>
  <Tab title="Linux">
    ```bash
    export DOCKER_IMAGE=intelanalytics/ipex-llm-inference-cpp-xpu:latest
    export CONTAINER_NAME=ipex-llm-inference-cpp-xpu-container

    sudo docker run -itd \
      --net=host \
      --device=/dev/dri \
      -v /path/to/models:/models \
      -e no_proxy=localhost,127.0.0.1 \
      --memory="32G" \
      --name=$CONTAINER_NAME \
      -e bench_model="mistral-7b-v0.1.Q4_0.gguf" \
      -e DEVICE=Arc \
      --shm-size="16g" \
      $DOCKER_IMAGE
    ```
  </Tab>
  <Tab title="Windows WSL">
    ```bash
    export DOCKER_IMAGE=intelanalytics/ipex-llm-inference-cpp-xpu:latest
    export CONTAINER_NAME=ipex-llm-inference-cpp-xpu-container

    sudo docker run -itd \
      --net=host \
      --device=/dev/dri \
      --privileged \
      -v /path/to/models:/models \
      -v /usr/lib/wsl:/usr/lib/wsl \
      -e no_proxy=localhost,127.0.0.1 \
      --memory="32G" \
      --name=$CONTAINER_NAME \
      -e bench_model="mistral-7b-v0.1.Q4_0.gguf" \
      -e DEVICE=Arc \
      --shm-size="16g" \
      $DOCKER_IMAGE
    ```

    <Note>
    On Windows WSL, enable `--net=host` by following the [Docker host network guide](https://docs.docker.com/network/drivers/host/#docker-desktop). The v6.1x WSL kernel is recommended to avoid GPU model-loading issues.
    </Note>
  </Tab>
</Tabs>

Enter the container:

```bash
docker exec -it ipex-llm-inference-cpp-xpu-container /bin/bash
```

### Run llama.cpp

```bash
cd /llm/scripts/
source ipex-llm-init --gpu --device $DEVICE
bash start-llama-cpp.sh
```

### Run Ollama

Start the Ollama server in the background:

```bash
cd /llm/scripts/
source ipex-llm-init --gpu --device $DEVICE
bash start-ollama.sh
```

Pull a model and run it:

```bash
cd /llm/ollama
./ollama pull llama2
./ollama run llama2
```

Test via the API:

```bash
curl http://localhost:11434/api/generate -d '{
  "model": "llama2",
  "prompt": "What is AI?",
  "stream": false
}'
```

### Open WebUI (optional)

Start Open WebUI after Ollama is running:

```bash
export DOCKER_IMAGE=ghcr.io/open-webui/open-webui:main
export CONTAINER_NAME=open-webui

docker run -itd \
  -v open-webui:/app/backend/data \
  -e PORT=8080 \
  --network=host \
  --name $CONTAINER_NAME \
  --restart always $DOCKER_IMAGE
```

Then visit [http://localhost:8080](http://localhost:8080). The default Ollama address is `http://localhost:11434`, configurable under **Admin Settings → Connections**.

## vLLM serving

The `ipex-llm-serving-xpu` image also includes the IPEX-LLM vLLM backend for high-throughput OpenAI-compatible serving.

### Pull the image

<Tabs>
  <Tab title="Intel Arc A770">
    ```bash
    docker pull intelanalytics/ipex-llm-serving-xpu:0.8.3-b19
    ```
  </Tab>
  <Tab title="Intel Arc BMG">
    ```bash
    docker pull intelanalytics/ipex-llm-serving-xpu:0.2.0-b2
    ```
  </Tab>
</Tabs>

### Start the container

```bash
export DOCKER_IMAGE=intelanalytics/ipex-llm-serving-xpu:latest
export CONTAINER_NAME=vllm-container

sudo docker run -itd \
  --net=host \
  --privileged \
  --device=/dev/dri \
  -v /path/to/models:/llm/models \
  -e no_proxy=localhost,127.0.0.1 \
  -e http_proxy=$HTTP_PROXY \
  -e https_proxy=$HTTPS_PROXY \
  --name=$CONTAINER_NAME \
  --shm-size="16g" \
  --entrypoint /bin/bash \
  $DOCKER_IMAGE

docker exec -it vllm-container /bin/bash
```

### Start the vLLM service

```bash
export USE_XETLA=OFF
export SYCL_CACHE_PERSISTENT=1
export SYCL_PI_LEVEL_ZERO_USE_IMMEDIATE_COMMANDLISTS=2
export FI_PROVIDER=shm
export TORCH_LLM_ALLREDUCE=0
export CCL_WORKER_COUNT=2
export CCL_ATL_TRANSPORT=ofi
export CCL_ZE_IPC_EXCHANGE=sockets
export CCL_ATL_SHM=1
export CCL_SAME_STREAM=1
export CCL_BLOCKING_WAIT=0
export LOAD_IN_LOW_BIT="fp8"
export IPEX_LLM_LOWBIT=$LOAD_IN_LOW_BIT
export VLLM_USE_V1=0

source /opt/intel/1ccl-wks/setvars.sh

numactl -C 0-11 python -m ipex_llm.vllm.xpu.entrypoints.openai.api_server \
  --port 8000 \
  --model "/llm/models/Qwen2.5-7B-Instruct/" \
  --served-model-name "Qwen2.5-7B-Instruct" \
  --trust-remote-code \
  --gpu-memory-utilization "0.95" \
  --device xpu \
  --dtype float16 \
  --enforce-eager \
  --load-in-low-bit $LOAD_IN_LOW_BIT \
  --max-model-len "2000" \
  --max-num-batched-tokens "3000" \
  --max-num-seqs "256" \
  --tensor-parallel-size "2" \
  --pipeline-parallel-size "1" \
  --block-size 8 \
  --distributed-executor-backend ray \
  --disable-async-output-proc
```

Alternatively, edit and run the built-in launch script:

```bash
bash /llm/start-vllm-service.sh
```

### Key parameters

| Parameter | Description |
|---|---|
| `--load-in-low-bit` | Quantization format: `sym_int4`, `asym_int4`, `fp6`, `fp8`, `fp16`. Default is `fp8`. |
| `--tensor-parallel-size` | Number of tensor parallel replicas (number of GPUs). |
| `--gpu-memory-utilization` | Fraction of GPU memory to allocate (0–1). Default is `0.9`. |
| `--max-model-len` | Model context length. Derived from model config if unset. |
| `--block-size` | vLLM KV cache block size. Set to `8` for a performance boost. |

### Test the service

```bash
curl http://localhost:8000/v1/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "Qwen2.5-7B-Instruct",
    "prompt": "San Francisco is a",
    "max_tokens": 128
  }'
```

## FastChat serving

FastChat is an open platform for training, serving, and evaluating LLMs. IPEX-LLM supports it as a backend using either its native worker or the vLLM worker.

### Start the container

```bash
export DOCKER_IMAGE=intelanalytics/ipex-llm-serving-xpu:latest

sudo docker run -itd \
  --net=host \
  --device=/dev/dri \
  --name=fastchat-container \
  -v /LLM_MODELS/:/llm/models/ \
  --shm-size="16g" \
  --entrypoint /bin/bash \
  $DOCKER_IMAGE

docker exec -it fastchat-container /bin/bash
```

### Start the FastChat service

```bash
export MODEL_PATH=/llm/models/YOUR_MODEL
export LOW_BIT_FORMAT=sym_int4
export CONTROLLER_HOST=localhost
export CONTROLLER_PORT=21001
export WORKER_HOST=localhost
export WORKER_PORT=21002
export API_HOST=localhost
export API_PORT=8000

# Use the default IPEX-LLM worker
bash /llm/start-fastchat-service.sh -w model_worker
```

To use the vLLM worker instead:

```bash
bash /llm/start-fastchat-service.sh -w vllm_worker
```

<Note>
The vLLM worker takes longer to start than the default `model_worker`.
</Note>
