title: "vLLM"

---

IPEX-LLM integrates with [vLLM](https://github.com/vllm-project/vllm) to provide high-throughput, continuous-batching LLM serving on Intel GPUs. The integration exposes a fully OpenAI-compatible HTTP API and supports tensor parallelism across multiple cards.

<Note>
  vLLM support is currently **Linux only**.
</Note>

**Supported models:**
- Qwen series
- Llama series
- ChatGLM series
- Baichuan series

## Installation

<Steps>
  <Step title="Install ipex-llm[xpu]">
    Follow the [Linux GPU installation guide](/quickstart/install-linux-gpu) to install prerequisites (GPU driver, oneAPI Base Toolkit), then install `ipex-llm[xpu]` and configure the runtime environment.

    Create a conda environment for vLLM:

    ```bash
    conda create -n ipex-vllm python=3.11
    conda activate ipex-vllm
    ```
  </Step>

  <Step title="Install vLLM">
    IPEX-LLM maintains a custom vLLM branch with Intel GPU support.

    ```bash
    conda activate ipex-vllm
    source /opt/intel/oneapi/setvars.sh

    pip install oneccl-bind-pt==2.1.300+xpu \
      --extra-index-url https://pytorch-extension.intel.com/release-whl/stable/xpu/us/

    git clone -b 0.5.4 https://github.com/analytics-zoo/vllm.git
    cd vllm
    pip install -r requirements-xpu.txt
    VLLM_TARGET_DEVICE=xpu python setup.py install

    pip install mpi4py fastapi uvicorn openai
    pip install gradio==4.43.0
    pip install ray
    ```
  </Step>
</Steps>

## Offline inference

For a quick test without starting the HTTP server, run offline inference directly in Python.

```python
from ipex_llm.vllm.xpu.engine import IPEXLLMClass as LLM
from vllm import SamplingParams

llm = LLM(
    model="YOUR_MODEL_PATH",
    device="xpu",
    dtype="float16",
    enforce_eager=True,
    load_in_low_bit="sym_int4",   # Options: sym_int4, fp6, fp8, fp8_e4m3, fp16
    tensor_parallel_size=1,
    trust_remote_code=True,
)

sampling_params = SamplingParams(temperature=0, max_tokens=128)
prompts = [
    "The capital of France is",
    "The future of AI is",
]

outputs = llm.generate(prompts, sampling_params)
for output in outputs:
    print(output.outputs[0].text)
```

Example output (`Baichuan2-7B-Chat` with `sym_int4`):

```
 Paris.
 The capital of France is Paris.
 bright, but it's not without challenges. As AI continues to evolve,
```

## Starting the vLLM server

The IPEX-LLM vLLM server implements the OpenAI API protocol. Start it with `ipex_llm.vllm.xpu.entrypoints.openai.api_server`.

```bash
model="YOUR_MODEL_PATH"
served_model_name="YOUR_MODEL_NAME"

python -m ipex_llm.vllm.xpu.entrypoints.openai.api_server \
  --served-model-name $served_model_name \
  --port 8000 \
  --model $model \
  --trust-remote-code \
  --gpu-memory-utilization 0.75 \
  --device xpu \
  --dtype float16 \
  --enforce-eager \
  --load-in-low-bit sym_int4 \
  --max-model-len 4096 \
  --max-num-batched-tokens 10240 \
  --max-num-seqs 12 \
  --tensor-parallel-size 1
```

<Tip>
  On Intel Arc A-Series with Linux kernel 6.2, set `export SYCL_PI_LEVEL_ZERO_USE_IMMEDIATE_COMMANDLISTS=1` before starting the server for a potential performance improvement.
</Tip>

### Key server parameters

| Parameter | Description |
|---|---|
| `--load-in-low-bit` | Quantization format: `sym_int4`, `fp6`, `fp8`, `fp8_e4m3`, `fp16` |
| `--gpu-memory-utilization` | Fraction of GPU memory to use (0–1, default: 0.9) |
| `--max-model-len` | Context length. Derived from model config if unspecified |
| `--max-num-batched-tokens` | Max tokens per forward pass. Reduce to lower peak memory during prefill |
| `--max-num-seqs` | Max concurrent sequences. Reduce to lower memory during decoding |
| `--tensor-parallel-size` | Number of GPUs to use for tensor parallelism |

<Note>
  Send a few warm-up requests before benchmarking — the first few requests trigger JIT kernel compilation.
</Note>

### Test the server

```bash
curl http://localhost:8000/v1/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "YOUR_MODEL_NAME",
    "prompt": "San Francisco is a",
    "max_tokens": 128,
    "temperature": 0
  }' | jq '.choices[0].text'
```

## Tensor parallelism

To distribute a model across multiple Intel GPUs, set `--tensor-parallel-size` to the number of cards and configure the OneCCL environment variables.

Example for two Arc A770 cards:

```bash
model="YOUR_MODEL_PATH"
served_model_name="YOUR_MODEL_NAME"

# OneCCL environment variables
export CCL_WORKER_COUNT=2
export FI_PROVIDER=shm
export CCL_ATL_TRANSPORT=ofi
export CCL_ZE_IPC_EXCHANGE=sockets
export CCL_ATL_SHM=1

python -m ipex_llm.vllm.xpu.entrypoints.openai.api_server \
  --served-model-name $served_model_name \
  --port 8000 \
  --model $model \
  --trust-remote-code \
  --gpu-memory-utilization 0.75 \
  --device xpu \
  --dtype float16 \
  --enforce-eager \
  --load-in-low-bit sym_int4 \
  --max-model-len 4096 \
  --max-num-batched-tokens 10240 \
  --max-num-seqs 12 \
  --tensor-parallel-size 2
```

<Note>
  For tensor parallel deployments, Intel recommends using the Docker image `intelanalytics/ipex-llm-serving-xpu`. See the [Docker deployment guide](/deployment/docker).
</Note>

## Docker deployment

Pull the prebuilt serving image:

```bash
# For Intel Arc A770
docker pull intelanalytics/ipex-llm-serving-xpu:0.8.3-b19

# For Intel Arc BMG
docker pull intelanalytics/ipex-llm-serving-xpu:0.2.0-b2
```

Start the container with GPU access:

```bash
export DOCKER_IMAGE=intelanalytics/ipex-llm-serving-xpu:latest
export CONTAINER_NAME=ipex-vllm-container

sudo docker run -itd \
  --net=host \
  --privileged \
  --device=/dev/dri \
  -v /path/to/models:/llm/models \
  -e no_proxy=localhost,127.0.0.1 \
  --name=$CONTAINER_NAME \
  --shm-size="16g" \
  --entrypoint /bin/bash \
  $DOCKER_IMAGE
```

Enter the container and start the service:

```bash
docker exec -it $CONTAINER_NAME /bin/bash
bash /llm/start-vllm-service.sh
```

## Benchmarking

Use the vLLM throughput benchmark script to measure performance:

```bash
conda activate ipex-vllm
source /opt/intel/oneapi/setvars.sh

wget https://huggingface.co/datasets/anon8231489123/ShareGPT_Vicuna_unfiltered/resolve/main/ShareGPT_V3_unfiltered_cleaned_split.json

export MODEL="YOUR_MODEL_PATH"

python3 benchmark_throughput.py \
  --backend vllm \
  --dataset ./ShareGPT_V3_unfiltered_cleaned_split.json \
  --model $MODEL \
  --num-prompts 1000 \
  --seed 42 \
  --trust-remote-code \
  --enforce-eager \
  --dtype float16 \
  --device xpu \
  --load-in-low-bit sym_int4 \
  --gpu-memory-utilization 0.85
```
