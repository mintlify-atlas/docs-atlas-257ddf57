IPEX-LLM provides first-class integrations for both [LangChain](https://www.langchain.com/) and [LlamaIndex](https://www.llamaindex.ai/), letting you build RAG pipelines, agents, and chat applications that run local LLMs on Intel GPU with low-bit quantization.

## LangChain

### Install LangChain dependencies

```bash
pip install -U langchain langchain-community
```

### Basic chat with `IpexLLM`

`IpexLLM` is a LangChain `LLM` class that loads any Hugging Face Transformer model with IPEX-LLM quantization.

```python
from langchain_community.llms import IpexLLM
from langchain_core.prompts import PromptTemplate
from langchain.chains import LLMChain

# Prompt template for LLaMA-2 instruct format
template = """<s>[INST] <<SYS>>\n    \n<</SYS>>\n\n{question} [/INST]"""
prompt = PromptTemplate(template=template, input_variables=["question"])

llm = IpexLLM.from_model_id(
    model_id="YOUR_MODEL_PATH",
    model_kwargs={
        "temperature": 0,
        "max_length": 64,
        "trust_remote_code": True,
        "device": "xpu",          # Use "cpu" for CPU inference
    },
)

chain = prompt | llm
output = chain.invoke("What is AI?")
print(output)
```

Run the example from the command line:

```bash
python chat.py -m YOUR_MODEL_PATH -q "What is AI?"
```

### Retrieval-augmented generation (RAG)

The RAG example builds a retrieval pipeline that stores documents in a Chroma vector store and uses IPEX-LLM for both embedding and generation.

Install additional dependencies:

```bash
pip install -U langchain langchain-community langchain-chroma sentence-transformers==3.0.1
```

```python
from langchain_community.llms import IpexLLM
from langchain_community.embeddings import IpexLLMBgeEmbeddings
from langchain_text_splitters import CharacterTextSplitter
from langchain_chroma import Chroma
from langchain import hub
from langchain_core.runnables import RunnablePassthrough
from langchain_core.output_parsers import StrOutputParser

# Source document
text_doc = """
IPEX-LLM is an LLM acceleration library for Intel CPU, GPU and NPU.
It provides seamless integration with Ollama, HuggingFace transformers,
LangChain, LlamaIndex, vLLM, FastChat, and more. 70+ models have been
optimized/verified on ipex-llm.
"""

# Split and embed the document
text_splitter = CharacterTextSplitter(chunk_size=1000, chunk_overlap=0)
texts = text_splitter.split_text(text_doc)

embeddings = IpexLLMBgeEmbeddings(
    model_name="EMBEDDING_MODEL_PATH",
    model_kwargs={"device": "xpu"},
    encode_kwargs={"normalize_embeddings": True},
)
retriever = Chroma.from_texts(
    texts,
    embeddings,
    metadatas=[{"source": str(i)} for i in range(len(texts))],
).as_retriever()

# Load the LLM
llm = IpexLLM.from_model_id(
    model_id="YOUR_LLM_MODEL_PATH",
    model_kwargs={
        "temperature": 0,
        "max_length": 512,
        "trust_remote_code": True,
        "device": "xpu",
    },
)

# Build the RAG chain
prompt = hub.pull("rlm/rag-prompt")

def format_docs(docs):
    return "\n\n".join(doc.page_content for doc in docs)

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt
    | llm
    | StrOutputParser()
)

response = rag_chain.invoke("What is IPEX-LLM?")
print(response)
```

Run the RAG example:

```bash
python rag.py \
  -m YOUR_LLM_MODEL_PATH \
  -e YOUR_EMBEDDING_MODEL_PATH \
  -q "What is IPEX-LLM?"
```

### Saving and loading a low-bit model

Use `save_low_bit` to quantize and persist a model, then load it later without re-quantizing:

```python
from langchain_community.llms import IpexLLM

# First run: quantize and save
llm = IpexLLM.from_model_id(
    model_id="YOUR_MODEL_PATH",
    model_kwargs={
        "load_in_low_bit": "sym_int4",
        "trust_remote_code": True,
        "device": "xpu",
    },
)
llm.model.save_low_bit("PATH_TO_SAVE_LOW_BIT_MODEL")

# Subsequent runs: load directly
llm = IpexLLM.from_model_id_low_bit(
    model_id="PATH_TO_SAVE_LOW_BIT_MODEL",
    model_kwargs={"trust_remote_code": True, "device": "xpu"},
)
```

<Note>
  `save_low_bit` saves model weights only. Copy the tokenizer files to the target directory or specify `tokenizer_id` during initialization.
</Note>

## LlamaIndex

### Install LlamaIndex packages

```bash
pip install llama-index-llms-ipex-llm[xpu]==0.1.8 \
  --extra-index-url https://pytorch-extension.intel.com/release-whl/stable/xpu/us/

pip install llama-index-embeddings-ipex-llm[xpu]==0.1.5 \
  --extra-index-url https://pytorch-extension.intel.com/release-whl/stable/xpu/us/
```

<Note>
  Installing `llama-index-llms-ipex-llm` or `llama-index-embeddings-ipex-llm` also installs `ipex-llm` and its dependencies. `IpexLLMEmbedding` currently supports Hugging Face BGE models only.
</Note>

### Using `IpexLLM` and `IpexLLMEmbedding`

```python
import torch  # must be imported first to avoid core dump
from llama_index.llms.ipex_llm import IpexLLM
from llama_index.embeddings.ipex_llm import IpexLLMEmbedding

# Define prompt formatting for the model (example: Zephyr)
def completion_to_prompt(completion):
    return f"<|system|>\n</s>\n<|user|>\n{completion}</s>\n<|assistant|>\n"

def messages_to_prompt(messages):
    prompt = ""
    for message in messages:
        if message.role == "system":
            prompt += f"<|system|>\n{message.content}</s>\n"
        elif message.role == "user":
            prompt += f"<|user|>\n{message.content}</s>\n"
        elif message.role == "assistant":
            prompt += f"<|assistant|>\n{message.content}</s>\n"
    if not prompt.startswith("<|system|>\n"):
        prompt = "<|system|>\n</s>\n" + prompt
    return prompt + "<|assistant|>\n"

# Load the LLM
llm = IpexLLM.from_model_id(
    model_name="YOUR_MODEL_PATH",
    tokenizer_name="YOUR_TOKENIZER_PATH",
    context_window=512,
    max_new_tokens=64,
    generate_kwargs={"temperature": 0.7, "do_sample": False},
    messages_to_prompt=messages_to_prompt,
    completion_to_prompt=completion_to_prompt,
    device_map="xpu",
)

# Load the embedding model
embed_model = IpexLLMEmbedding(
    model_name="BAAI/bge-small-en",
    device="xpu",
)
```

### RAG with PostgreSQL vector store

This example indexes a PDF document into PostgreSQL (using pgvector) and builds a query engine over it.

**Install additional dependencies:**

```bash
pip install llama-index-readers-file==0.1.33
pip install llama-index-vector-stores-postgres==0.1.14
pip install pymupdf psycopg2
```

```python
import torch  # import first
from llama_index.embeddings.ipex_llm import IpexLLMEmbedding
from llama_index.llms.ipex_llm import IpexLLM
from llama_index.vector_stores.postgres import PGVectorStore
from llama_index.readers.file import PyMuPDFReader
from llama_index.core.node_parser import SentenceSplitter
from llama_index.core.vector_stores import VectorStoreQuery
from llama_index.core.query_engine import RetrieverQueryEngine
from llama_index.core.retrievers import BaseRetriever
import psycopg2

# Initialize the embedding model
embed_model = IpexLLMEmbedding(model_name="BAAI/bge-small-en", device="xpu")

# Initialize the LLM
llm = IpexLLM.from_model_id(
    model_name="YOUR_MODEL_PATH",
    tokenizer_name="YOUR_TOKENIZER_PATH",
    context_window=512,
    max_new_tokens=32,
    generate_kwargs={"temperature": 0.7, "do_sample": False},
    device_map="xpu",
)

# Set up the PostgreSQL vector store
conn = psycopg2.connect(dbname="postgres", host="localhost",
                        password="YOUR_PASSWORD", port="5432", user="YOUR_USER")
conn.autocommit = True
with conn.cursor() as c:
    c.execute("DROP DATABASE IF EXISTS example_db")
    c.execute("CREATE DATABASE example_db")

vector_store = PGVectorStore.from_params(
    database="example_db", host="localhost",
    password="YOUR_PASSWORD", port="5432", user="YOUR_USER",
    table_name="documents", embed_dim=384,
)

# Load and index documents
loader = PyMuPDFReader()
documents = loader.load(file_path="data/llama2.pdf")
splitter = SentenceSplitter(chunk_size=1024)

nodes = []
for doc in documents:
    for chunk in splitter.split_text(doc.text):
        from llama_index.core.schema import TextNode
        node = TextNode(text=chunk)
        node.metadata = doc.metadata
        node.embedding = embed_model.get_text_embedding(chunk)
        nodes.append(node)

vector_store.add(nodes)

# Query
query_str = "How does Llama 2 compare to other open-source models?"
query_embedding = embed_model.get_query_embedding(query_str)
query_result = vector_store.query(
    VectorStoreQuery(query_embedding=query_embedding, similarity_top_k=2)
)
print(query_result.nodes[0].get_content())
```

Run the full RAG example from the command line:

```bash
python rag.py \
  -m YOUR_MODEL_PATH \
  -t YOUR_TOKENIZER_PATH \
  -u YOUR_DB_USER \
  -p YOUR_DB_PASSWORD \
  -q "How does Llama 2 compare to other open-source models?"
```

Example output:

```
The comparison between Llama 2 and other open-source models is complex and depends
on various factors such as the specific benchmarks used, the model size, and the
task at hand. In terms of performance on the benchmarks provided, Llama 2 outperforms
other open-source models on most categories...
```
